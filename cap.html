Redux

Redux es otro manejador de estados que tenemos disponible en Flutter al igual que Provider y en esta sección, vamos a conocer cómo emplearlo.

Propósito de los manejadores de estados

Cuando desarrollamos aplicaciones del lado cliente complejas dividimos una interfaz grande en pequeñas partes llamadas componentes cada componente puede tener un estado que no es más que datos que posea una interfaz en un momento dado esos componentes a su vez pueden estar compuestos de otros componentes formando lo que se conoce como un árbol de componentes pero puede surgir una dificultad y es que si tienes un componente que debe pasar datos a otros componentes hijos o estos a su vez deben pasar datos a sus componentes hijos o necesito que múltiples componentes puedan acceder al mismo estado la solución sencilla de pasar un dato de un componente a otro puede hacer que mi código se convierte en algo complicado de mantener y hacer testing es decir tener descentralizado el estado es difícil de manejar así que es aquí donde entran las bibliotecas de manejo de estado.

Estas bibliotecas para manejar los estados, permiten mantener un estado en un solo lugar por aparte de todo nuestro árbol de componentes es decir permiten tener un estado centralizado de esta forma no importa en qué componente estemos desarrollando siempre va a haber un solo lugar en donde podamos acceder al estado lo que nos evita estar pensando en la jerarquía de los componentes entre estas bibliotecas de manejo de estado.

Componentes de Redux

Redux consta de varios componentes o partes como veremos a continuación.

Store

El store es el único repositorio en donde se almacenan y controlan el acceso a los datos; puedes verlo como la base de datos de la aplicación, pero, estos datos ya los tenemos en propiedades como arrays, números, objetos, entre otros.

Este repositorio que llamaremos cómo store, es inmutable en esta capa, es decir, no lo podemos modificar de manera directa, pero, si no lo modificamos no tendría sentido emplear un gestor de estado, ya que la idea de los mismos es que podamos gestionar los datos y esto implica poder modificarlos; cuando queramos modificar el store, debemos de hacerlo mediante una interfaz, que usualmente empleamos cuando se ejecuta un evento como podría ser un evento click, submit entre otros.

Acciones

Una acción es sólo un objeto que contiene información y le dice a redux qué operación se va a ejecutar sobre el store, es decir, las típicas operaciones CRUD para agregar, modificar, eliminar o simplemente obtener datos.

Las acciones pueden o no estar acompañado de un dato, es decir, un parámetro, que son los cambios que queremos registrar o cambiar en el store, a continuación, un ejemplo de una acción:

Estas acciones realizan los cambios en el estado, es decir, como puedes apreciar en el código anterior, hacen el get/set de los datos en base a nuestra lógica de negocios.

Reducer

Un reducer es solo una función que espera dos parámetros:

  1. El estado actual, para saber qué datos tiene la interfaz hasta ese momento.
  2. El action para saber qué datos debe ejecutar, el action usualmente es empleado en un condicional para ejecutar alguna porción de código y modificar el estado en consecuencia.

Implementación

Finalmente, explicada ya de manera teórica el funcionamiento de redux, vamos a pasar a la implementación.

        store.state.products.indexWhere((p) => p.id == pString['product_id']);
